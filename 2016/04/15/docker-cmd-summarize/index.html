docker pull

说明：获取网络镜像到本地


当不指定TAG时，默认为：latest
示例：

docker images

说明：显示本地已有的镜像

镜像的 ID 唯一标识了镜像。

docker run

说明：使用已下载到本地的镜像启动一个新容器

IMAGE可以为镜像名或者镜像ID。
示例：

在执行 docker run 的时候如果添加 –rm 标记，则容器在终止后会立刻删除。注意，–rm 和 -d 参数不能同时使用。

docker logs

说明：获取容器输出的信息，适用于守护态运行docker容器。

docker stop

说明：停止一个正在启动的容器。

docker start

说明：启动一个已经停止的容器。

docker restart

说明：将一个运行态的容器终止，然后再重新启动它。

docker attach

说明：在使用 -d 参数时，容器启动后会进入后台。 某些时候需要进入容器进行操作就可以用它。

docker commit

说明：将修改后的容器提交保存为一个新的镜像。

示例：

docker tag

说明:修改镜像标签

示例：

docker push

说明:自己创建的本地镜像上传到远程仓库中来共享。

docker save

说明:将镜像导出成本地文件

示例：

docker load

说明:将镜像本地文件加载成为本地镜像

示例：

docker ps

说明：列出对应的容器

示例：

docker export

说明：导出本地某个容器为快照文件

docker import

说明：将容器快照文件导入为本地容器或者本地镜像。与docker load的区别在于容器快照文件将丢弃所有的历史记录和元数据信息,而镜像存储文件将保存完整记录，体积也要大。

示例：

docker rm

说明：移除容器

示例：

docker rmi

说明:移除本地镜像，移出前需要先移除到镜像对应的容器。

清理所有未打过标签的本地镜像：docker rmi $(docker images -q -f "dangling=true")

docker inspect

说明:查看容器或者镜像的详情信息。

docker port

说明：来查看当前映射的端口配置，也可以查看到绑定的地址。

观点仅代表自己，期待你的留言。
